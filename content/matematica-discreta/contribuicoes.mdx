O MacDocs é feito de arquivos **MDX** — texto normal, com fórmulas em LaTeX
e alguns componentes prontos (`Question`, `Answer`, `Callout` etc). Não é
preciso saber programar para contribuir: encontrou um erro numa resolução,
quer sugerir uma questão nova ou tem uma prova resolvida para compartilhar?
Toda contribuição de **conteúdo** — não só de código — é bem-vinda.

## Sem usar o GitHub

Se preferir não mexer com Git, mande o material direto para o autor:

- Foto ou PDF da prova/lista, com as respostas.
- Ou o texto da questão e da resposta, do jeito que for mais fácil de
  escrever.

Isso é o suficiente. A transcrição para o formato do site (como foi feito
nas provas publicadas) pode ser feita por quem mantém o repositório.

## Usando o GitHub

Para quem tem conta no GitHub, duas formas, da mais simples à mais
completa:

### 1. Editar uma página existente

Toda página tem um link **"Editar esta página no GitHub"** no rodapé. Ele
abre o arquivo `.mdx` correspondente direto no editor do GitHub — clicando
em editar, o GitHub cria um fork automaticamente e, ao salvar, oferece
a opção de abrir um Pull Request. Não precisa clonar nada.

### 2. Adicionar uma página nova (nova questão, prova ou tópico)

<StepByStep>

<Step title="Faça um fork">
Fork do repositório [mac-docs](https://github.com/dariomatias-dev/mac-docs).
</Step>

<Step title="Crie o arquivo">
No fork, use **Add file → Create new file** dentro de `content/` para criar
o `.mdx`. Siga a estrutura de uma página parecida como referência — por
exemplo, uma [prova resolvida](/docs/matematica-discreta/avaliacoes/2023.1/prova-1)
para questões, ou qualquer página de tópico para conteúdo teórico.
</Step>

<Step title="Preencha o conteúdo">
Frontmatter (`title`, `description`, `order`) e o conteúdo em
Markdown/LaTeX.
</Step>

<Step title="Abra o Pull Request">
Descreva a origem do material (disciplina, semestre, professor).
</Step>

</StepByStep>

<Callout type="tip" title="Dica">
  Não se preocupe em acertar a formatação perfeitamente. É mais importante o
  conteúdo estar correto e fiel à fonte — ajustes de estilo podem ser feitos
  na revisão do Pull Request.
</Callout>

## O que mais ajuda

- Correções de erros em resoluções ou enunciados.
- Novas questões resolvidas (provas, listas, exercícios de livro).
- Sugestões de tópicos que faltam.

Dúvidas ou material para enviar sem GitHub: abra uma
[issue](https://github.com/dariomatias-dev/mac-docs/issues) ou entre em
contato diretamente com o autor.
